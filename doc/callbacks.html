Callbacks


For more fine-grained control, libcurl allows a number of callbacks to be associated with each connection.  In pycurl, callbacks are defined using the setopt() method for Curl objects with options WRITEFUNCTION, READFUNCTION, HEADERFUNCTION, PROGRESSFUNCTION, PASSWDFUNCTION or DEBUGFUNCTION.  Options correspond to the libcurl options.  A valid callback in pycurl is a regular function, a class method or an extension type function.

The signature of each callback used in pycurl is as follows:

WRITEFUNCTION(string) -> number of bytes written

READFUNCTION(number of bytes to read) -> string

HEADERFUNCTION(string) -> number of bytes written

PROGRESSFUNCTION(download total, download now, upload total, upload now) -> status

PASSWDFUNCTION(prompt string, max number of bytes to read) -> string

DEBUGFUNCTION(debug type integer, debug message) -> None
